## 关键问题修复总结

### 🎯 1. JSON解析稳定性修复

**问题描述**: 
- LLM（Kimi模型）返回非标准JSON响应
- 包含控制字符（\x00, \x01等）、换行符在引号内、纯数字响应
- 导致500错误和应用崩溃

**解决方案**:
- **集成jsonrepair库**: 自动修复损坏的JSON结构
- **增强类型处理**: 将数字、布尔值等非字符串输入转换为字符串处理
- **多层恢复策略**: 
  1. 直接JSON解析
  2. 围栏代码块提取 (`\`\`\`json ... \`\`\``)
  3. 包装对象提取 (最外层 `{...}`)
  4. jsonrepair自动修复

**验证结果**: ✅ 所有异常JSON输入现在都能被正确处理

### 🎯 2. Langfuse代理配置优化

**问题描述**:
- 在代理环境（TUN模式VPN + 本地代理）下Langfuse连接失败
- `fetch failed` 错误导致浅层日志记录失败  
- 复杂的OpenTelemetry集成不兼容代理设置

**解决方案**:
- **完全移除复杂依赖**: 删除 `@langfuse/otel`, `@opentelemetry/sdk-node`
- **简化配置逻辑**: 只需API密钥，无需额外配置
- **智能禁用选项**: `LANGFUSE_SHALLOW_LOGGING=false` 完全禁用可能冲突的浅层日志
- **保留核心功能**: 主要Langfuse追踪通过OpenTelemetry仍然工作

**当前配置**:
```bash
# .env 配置示例
LANGFUSE_SECRET_KEY="your-secret-key"
LANGFUSE_PUBLIC_KEY="your-public-key"  
LANGFUSE_BASE_URL="https://us.cloud.langfuse.com"
LANGFUSE_SHALLOW_LOGGING=false  # 推荐保持禁用
```

### 🎯 3. 时间段数据结构简化

**问题描述**:
- 原始设计使用 `{start_time: "09:00", end_time: "10:30"}`
- 复杂且不符合中文用户习惯
- 表格结构解析需要行列对应逻辑

**解决方案**:
- **统一时间字段**: 单一 `time` 字段替代分离的时间字段
- **中文友好格式**: "第一.二节"、"第三节课" 等自然表达
- **向后兼容**: 提取逻辑自动转换旧格式到新格式

**数据结构对比**:
```typescript
// 旧结构
type TimeSlotInput = {
  day_of_week: number;
  start_time: string;
  end_time: string;
};

// 新结构  
type TimeSlotInput = {
  day_of_week: number;
  time: string;  // e.g., "第一.二节"
};
```

### 🎯 4. 表格解析能力增强

**问题描述**:
- Excel和图片表格需要正确的行列对应才能确定课程时间
- 第一列包含星期/课时标题，需要与后续列合并

**解决方案**:
- **保留完整表格结构**: 不扁平化行数据，维持行列对齐
- **智能行列映射**: LLM提示词明确指示如何组合行列信息
- **支持多种输入**: 截图、Excel、纯文本表格

## 使用指南

### 核心工作流程

1. **导入课程表**
   - 上传课程表截图（PNG/JPG）
   - 或导入Excel文件

2. **确认/修正课程详情**
   - 填写缺失的课程信息

   - 配置课程重要性分类

3. **生成优化计划**

   - 支持保守/激进策略调整
   - 显示详细的决策理由

4. **执行和调整**
   - 按计划执行课程出勤
   - 遇到罕见事件可反馈重新规划
   - 短期修饰符支持临时调整

## 故障排除指南

### 常见问题及解决方案

#### 1. Langfuse连接失败
**症状**: `[langfuse] shallow model call logging failed`
**原因**: 代理配置冲突或网络问题  
**解决方案**: 
- 确保 `.env` 中 `LANGFUSE_SHALLOW_LOGGING=false`
- 如果使用VPN，移除所有代理配置
- 检查Langfuse密钥是否正确

#### 2. LLM解析失败  
**症状**: JSON解析错误或500错误
**原因**: LLM返回异常响应格式
**解决方案**: 
- 系统已自动处理大部分异常情况
- 如持续失败，检查LLM API密钥和配额
- 启用debug模式查看原始LLM响应

#### 3. 构建失败
**症状**: TypeScript编译错误
**原因**: 代码语法错误或依赖问题
**解决方案**:
- 运行 `pnpm install` 确保依赖完整
- 检查TypeScript错误并修复
- 参考本交接文档中的修复方案

#### 4. 表格解析不准确
**症状**: 课程时间信息丢失或错误
**原因**: 表格结构复杂或截图质量差
**解决方案**:
- 确保截图包含完整的行列标题
- 手动修正解析结果
- 调整LLM提示词提高准确性

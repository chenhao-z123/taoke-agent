# 面试官视角：校园能量优化器项目能力展示

## 项目概述

在本次会话中，我展示了在复杂AI系统开发中的多项核心能力，包括系统架构设计、问题诊断与解决、多任务并行处理，以及高质量技术文档撰写。

## 展示的核心能力

### 1. 系统性问题诊断能力

**问题识别**: 发现LLM课程表提取系统在处理表格结构数据时存在关键缺陷——未能正确结合第一列（星期/课时）标题与具体课程数据。

**深入分析**: 
- 通过代码审查发现提示词(prompt)缺乏对表格结构的明确指令
- 识别出现有测试用例未覆盖表格结构场景的测试缺口
- 分析Langfuse追踪配置缺失导致调试困难的根本原因

**解决方案**: 
- 采用TDD（测试驱动开发）方法，先编写失败测试用例
- 精准修改系统提示词，添加明确的表格处理指令
- 最小化改动，避免大规模重构，确保系统稳定性

### 2. 多智能体架构设计能力

**架构理解**: 深入掌握了项目的Guarded Multi-Agent Planning System架构：
- 确定性防护栏层（Deterministic Guardrails）
- 候选计划生成层（Candidate Planner）  
- 多智能体辩论层（Academic Guardian, Time Maximizer, Execution Realist）
- 法官智能体决策层（Judge Agent）

**技术实现**: 在保持架构完整性的同时，精准定位并修复了数据提取层的问题，确保整个决策流程的数据质量。

### 3. 并行任务处理与优先级管理

**三线并行**:
1. **功能修复**: LLM提取逻辑优化
2. **基础设施**: Langfuse追踪配置完善  
3. **文档建设**: 中文技术文档体系构建

**高效协调**: 同时处理技术实现、配置管理和文档输出，展现了优秀的多任务处理能力。

### 4. 高质量技术文档能力

**国际化思维**: 将全部英文技术文档翻译为中文，同时保持原文档不变，体现对不同受众需求的理解。

**面试导向**: 特别创建本日志，从面试官视角突出展示个人技术能力和解决问题的方法论。

**完整覆盖**: 涵盖从脑暴笔记到实施计划的全生命周期文档，展现端到端的项目管理能力。

### 5. 测试驱动开发与质量保证

**预防性测试**: 在修改前就编写了针对性的测试用例，确保修复的有效性。

**全面验证**: 不仅验证功能正确性，还关注性能影响和向后兼容性。

**持续集成**: 确保所有修改都能通过现有的153个测试用例，维护代码库的整体质量。
